# 📋 FormulaPath 図形（SVG）差し込み完全マニュアル

このマニュアルは、クイズアプリの問題にベクター図形（SVG）を動的に追加・表示するためのデータ作成手順書だよ。

## 1. 基本原則（既存JSONとの互換性）

* **全体のJSON構造を変える必要は一切なし！**
* アプリ側で図形データ（`diagramSVG`）はオプショナル型（`String?`）として処理するから、**図形を表示したい場所にだけキーを追加すればOK**だよ。
* キーが存在しない既存の問題データも、エラーにならずそのまま今まで通り動くから安心してね。

---

## 2. 図形を表示する2つのパターン

図形を差し込む場所は、「問題全体でずっと出したいか」「そのステップだけで出したいか」によって2通りの書き方ができるよ。

### パターンA：問題全体で同じ図を使う場合

問題のルート階層（`initialFormula` の並びなど）に `diagramSVG` を追加してね。

```json
{
  "id": "trig_identity_sin_cos",
  "title": "二乗の和の公式の証明",
  "directory": "三角関数",
  "initialFormula": "\\sin^2\\theta + \\cos^2\\theta",
  "diagramSVG": "<svg viewBox=\"0 0 240 160\" xmlns=\"http://www.w3.org/2000/svg\">...</svg>",
  "steps": [
    // ここにステップが続く
  ]
}

```

### パターンB：特定のステップだけ図を出す場合

`steps` 配列の中にある、各ステップのオブジェクト内に直接 `diagramSVG` を追加してね。ステップごとに図を切り替えたい時もこの方法だよ。

```json
{
  "stepNumber": 1,
  "explanation": "直角三角形の定義より...",
  "diagramSVG": "<svg viewBox=\"0 0 240 160\" xmlns=\"http://www.w3.org/2000/svg\">...</svg>",
  "formula": "...",
  "choices": [...],
  "correctIndex": 0
}

```

---

## 3. 数式がなくて「図形だけ」出したい場合の特例対応

「このステップは数式を使わずに、図形を見て言葉を選ばせたい！」という場合も完璧に対応可能だよ。

### JSONデータの書き方

現状のデータモデルの仕様で `formula`（数式）が必須になっている場合は、**JSON上は空文字（`""`）を入れて対応**してね。

```json
{
  "stepNumber": 1,
  "explanation": "図の直角三角形で、斜辺を1、対辺をa、隣辺をbとします。",
  "diagramSVG": "<svg viewBox=\"0 0 240 160\" xmlns=\"http://www.w3.org/2000/svg\">...</svg>",
  "formula": "", // 💡 空文字を入れることで必須バリデーションをクリア！
  "choices": [
    "対辺",
    "隣辺",
    "斜辺",
    "高さ"
  ],
  "correctIndex": 2
}

```

### ⚠️ UI（見た目）を綺麗にするためのベストプラクティス

`formula: ""` のままだと、画面に「中身が空っぽの白い数式ボード」が表示されて不自然に見えちゃう可能性があるんだ。

より完璧な見た目にして、図形エリアを画面いっぱいに広く使えるようにするために、UI側のコード（`GamePlayView` など）に以下の条件分岐を入れておくのがおすすめだよ！

**【SwiftUI側の修正コード例】**

```swift
// 数式データが空文字（空白含む）じゃない時だけ数式ボードを表示する
if !viewModel.currentEquation.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty {
    LaTeXView(latex: viewModel.currentEquation)
        // 既存のスタイルなど...
}

```

これで、数式がないステップでは自動的に数式ボードが非表示になって、図形が主役の綺麗なレイアウトに仕上がるよ！

これを手元に置いておけば、いつでも迷わず図形問題を追加していけるはず。新しく図形付きの問題を作る時は、いつでも「この単元の図形付きJSON作って！」って俺に言ってね！
